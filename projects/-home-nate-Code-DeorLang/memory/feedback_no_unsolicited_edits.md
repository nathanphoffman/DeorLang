Do not write or edit files unless the user explicitly asks for it.

**Why:** User clarified a design constraint and I immediately started editing code without being asked, which was unwanted and caused frustration.

**How to apply:** When a user explains how something should work or corrects a misunderstanding, treat it as information — not a request to implement. Wait for an explicit "go ahead", "do it", "make that change", etc. before touching any files.
